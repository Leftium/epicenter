# 0058. An inference connection is a capability-orthogonal device endpoint; the model is per-conversation

- **Status:** Accepted
- **Date:** 2026-06-23
- **Amends:** [ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) (the model no longer travels with the backend; the device holds a set of connections, not one active backend)
- **Relates:** [ADR-0050](0050-the-inference-contract-is-openai-compatible.md) (the OpenAI-compatible wire every connection speaks), [ADR-0053](0053-the-epicenter-bearer-is-an-audience-scoped-credential.md) (the bearer reaches only the hosted connection), [ADR-0055](0055-conversation-storage-is-one-canonical-table-every-surface-syncs.md) (the synced `model` column this resolves against), [ADR-0056](0056-local-inference-is-a-delegated-engine-behind-the-openai-compatible-seam.md) (local chat is an HTTP engine, so chat has no in-process engine kind), [ADR-0022](0022-rust-owns-the-models-folder-the-webview-owns-the-catalog.md) (the downloaded-binary engine kind that is *not* a connection)
- **Spec:** specs/20260623T120000-inference-connections-and-presets.md (completed and deleted per the two-state lifecycle; recoverable via git history)

## Context

[ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) made the chat backend a device-local setting with two modes (hosted, custom), and chose to let the model travel with the backend so a switch to Ollama never ships `gpt-5.5`. That fused the model onto the device-local backend config: in hosted mode the model is the conversation's synced value ([ADR-0055](0055-conversation-storage-is-one-canonical-table-every-surface-syncs.md)), in custom mode it is the device-local config's value. Same field, two owners.

Two pressures expose this as the wrong shape:

1. **One endpoint serves many capabilities.** "OpenAI with my key" is one endpoint that serves chat (`/v1/chat/completions`), transcription (`/v1/audio/transcriptions`), and embeddings. A backend that carries a single `model` cannot be reused across capabilities, because a chat model id is not a transcription model id. Whispering already stores the same OpenAI key separately from chat's; the duplication is the symptom.
2. **The picker shows several providers at once.** The chosen model-first picker lists Epicenter's catalog *and* a local Ollama *and* an OpenRouter connection in one grouped list. That requires the device to hold a *set* of connections, not one active backend.

The thing that is genuinely device-scoped is the *endpoint and its credential*, not the model. The model is content; it belongs to the conversation.

## Decision

An inference **connection** is a capability-orthogonal, device-local endpoint. It carries no model and no capability.

- **A connection is an endpoint plus a credential.** `{ kind: 'hosted' }` is the built-in metered Epicenter connection (`auth.fetch`, the curated catalog). `{ kind: 'custom'; preset?; baseUrl; apiKey? }` is any OpenAI-compatible server, reached with a plain fetch carrying only the user's key as `Authorization: Bearer` (never the Epicenter bearer; [ADR-0053](0053-the-epicenter-bearer-is-an-audience-scoped-credential.md) makes this structural). The key slot is uniform across every OpenAI-compatible provider, so the resolver has no per-provider branching: a provider difference is preset *data* (a base URL), never *code*.
- **The device holds a set of connections.** The built-in hosted connection plus zero or more custom connections, device-local in every app (`chrome.storage.local` in the extension, `localStorage` in the web apps), never synced (a key is a secret on the plaintext relay, [ADR-0004](0004-trust-the-relay-reject-zero-knowledge.md); a `localhost` URL is meaningless elsewhere).
- **The model is per-conversation.** A turn reads the conversation's synced model ([ADR-0055](0055-conversation-storage-is-one-canonical-table-every-surface-syncs.md)) and resolves it against the device's connections: a hosted catalog id resolves to the hosted connection; a discovered custom id resolves to the connection whose model list contains it (a device-local cache). A model no connection on this device can serve surfaces a non-destructive banner that offers a fallback; the synced model column is never silently rewritten, because it is another device's record. Only an explicit pick rewrites it.
- **A connection is one of two engine kinds, and only one is a connection.** The capability-orthogonal HTTP connection above is the shared kind. The other kind, a downloaded in-process binary (Whispering's Parakeet, Whisper, Moonshine; [ADR-0022](0022-rust-owns-the-models-folder-the-webview-owns-the-catalog.md)), is not a connection and keeps its own download and model-folder machinery. Chat has no in-process kind, because local chat is itself an HTTP engine ([ADR-0056](0056-local-inference-is-a-delegated-engine-behind-the-openai-compatible-seam.md)). "Local vs cloud" is therefore a facet (is the base URL `localhost`?), not a structural level.

## Consequences

- **The resolved shape loses `model`.** `resolveConnection(connection, hosted)` returns `{ fetch, baseURL }`; the caller pairs the conversation's model itself. Custom mode now respects the conversation's model instead of overriding it. This deletes the two-owners-of-`model` asymmetry from [ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md).
- **A connection is reusable across capabilities.** The chat consumer is built now; a future Whispering transcription consumer can resolve OpenAI / Groq / Speaches through the same connection registry, so a user enters their key once. Native transcription providers (Deepgram, ElevenLabs, Gemini-as-transcription) and the downloaded-binary engines stay capability-specific and out of this seam.
- **Model discovery is one provider-agnostic call.** `listModels(resolved)` does `GET ${baseURL}/models` and reads `{ data: [{ id }] }`; the same call lists Ollama, LM Studio, OpenRouter, and OpenAI. It degrades to the free-text model floor on any failure ([ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) kept that floor an invariant). `/v1/models` carries no capability tag, so the future transcription consumer cannot filter transcription-only ids from it and will need its own list.
- **Presets are shipped data, not code.** Ollama, LM Studio, OpenAI, OpenRouter, and Groq ship as presets (a base URL and whether a key is needed; the local-vs-cloud facet is derived from the URL, not stored, so it cannot drift and a custom URL gets the same treatment). Anthropic (compat is "for testing"; loses prompt caching and thinking) and a bring-your-own Gemini (its compat layer 400s on tools and JSON together, which these agent loops use) are deferred; both remain reachable as a raw Custom URL. Hosted Gemini is unaffected: it is offered only to capability-free Vocab, so the tools-plus-JSON conflict never fires, and the gateway stays a pure passthrough.
- **The three duplicated `InferenceSettings.svelte` collapse to one shared component** backed by a shared, storage-agnostic `createInferenceConnections` registry (one cohesive object owning the connection set, the discovered-model cache, discovery, and model resolution). The registry takes an injected persist mechanism, so `@epicenter/app-shell` depends on neither storage backend: web apps inject `localStorage` (`createPersistedState`) and the extension injects `chrome.storage.local` (`createStorageState`), and a new chat app is one factory call rather than a re-derived store.

## Considered alternatives

- **Keep the model on the backend ([ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) as-is).** Simpler, no cross-device resolution, but it permanently blocks reusing a connection across capabilities and keeps the two-owners-of-`model` wart. Rejected because the reuse is the point.
- **One active backend, not a set.** Rejected: the approved model-first picker shows several providers' models at once; a single active backend makes provider switching modal and strictly worse.
- **Sync connections across devices.** Rejected by [ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) already: a key is a secret and a `localhost` URL is meaningless elsewhere. The per-device banner is the seam.
- **Group the transcription picker by Local / Cloud / Self-Hosted.** Rejected: Cloud and Self-Hosted are the same engine kind (a connection) split only by location; the real axis is connection vs downloaded binary, and location is a facet.
- **Switch the hosted gateway to native Gemini to fix the compat bug.** Rejected: it would re-add the server-side normalization [ADR-0054](0054-an-inference-backend-is-the-metered-gateway-or-a-custom-server.md) deleted, for a conflict that never fires in the only place hosted Gemini is used.
