# Whispering v7.11.1 - Recording Markdown Export

Whispering now ships a one-shot Markdown export. Open Recording settings, click "Export markdown…", pick a folder, and Whispering writes every recording row's transcript to that folder as `<id>.md`. The transcript is the file body; the other recording fields are YAML frontmatter.

Snapshot semantics: the exported files reflect your recordings at the moment you clicked Export. Edits you make later in Whispering do not update the files. Run the export again whenever you want a fresh copy.

Why one-shot and not live sync:
- A live mirror needs an observer, a coalescing queue, dispose lifecycle, and a "did it write yet" mental model. A one-shot click avoids all of that.
- The exported files are derived output, not a second source of truth. Editing a generated `.md` file does not update Whispering state.

Legacy files: Whispering used to write Markdown sidecars into the appdata `recordings/` folder. Those files, if present, are left in place. You can delete them manually if you no longer need them.
